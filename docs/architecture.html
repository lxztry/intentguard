架构设计

IntentGuard 核心组件与验证流程

数据流

📝
需求解析器 — 将自然语言需求转换为 Constraint 列表
↓
🔧
代码分析器 — 解析代码为 AST / 正则匹配，提取函数/调用/条件
↓
⚙️
约束验证引擎 — 多层级验证（Static / Semantic / Deep）
↓
📊
报告生成器 — Text / JSON / Markdown 格式输出

项目结构

intentguard/
├── src/
│   ├── core/
│   │   ├── constraint.py     # 约束数据模型
│   │   ├── engine.py         # 多层级验证引擎
│   │   └── report.py         # 报告生成器
│   ├── parser/
│   │   └── pattern_parser.py # 自然语言 → 约束
│   ├── analyzers/
│   │   ├── python_analyzer.py   # Python AST 分析
│   │   ├── javascript_analyzer.py # JS/TS 正则解析
│   │   └── go_analyzer.py       # Go 正则解析
│   ├── llm/
│   │   ├── factory.py        # LLM 工厂（多提供商）
│   │   └── semantic.py       # LLM 语义验证
│   └── cli/
│       └── main.py           # CLI 入口
├── tests/                      # 测试套件
├── vscode-extension/           # VSCode 插件
└── docs/                       # 文档站

核心组件

组件职责
RequirementParser解析自然语言需求为结构化 Constraint 对象
PythonAnalyzerPython AST 解析，提取函数/调用/条件
JavaScriptAnalyzerJS/TS 正则解析（函数定义/调用/关键字）
GoAnalyzerGo 正则解析（函数/导入/错误处理）
LLMFactory统一接口：SiliconFlow / DeepSeek / OpenAI / Ollama
SemanticVerifierLLM 深层语义验证
EnhancedConstraintEngine三级验证：static / semantic / deep
ReportGeneratorText / JSON / Markdown 多格式报告

约束类型

类型说明示例
MUST必须实现"必须发送短信通知"
MUST_NOT禁止出现"禁止直接操作数据库"
CONDITIONAL条件触发"当订单失败时必须重试3次"
EXISTS必须存在"必须有权限校验"
LTE上限"响应时间不超过200ms"
GTE下限"至少重试3次"

级别方式速度精度LLM
