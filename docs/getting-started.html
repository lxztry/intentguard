快速开始

5 分钟上手 IntentGuard

环境要求

  • Python 3.10+
  • pip
  • （可选）LLM 验证需要 API Key

安装

git clone https://github.com/lxztry/intentguard.git
cd intentguard
pip install -r requirements.txt

基本用法

1

准备需求文件

创建 requirements.txt，每行一条自然语言需求：

用户下单后必须发送短信通知
每次API调用必须记录日志
禁止直接操作数据库，必须通过Repository层
2

验证代码文件

python -m src.cli.main verify -f path/to/code.py -r requirements.txt
3

查看结果

验证级别

IntentGuard 提供三级验证，按速度和精度权衡：

--level static（默认）
纯静态模式匹配，无需 LLM，速度最快，适合 CI/CD 快速检查。
--level semantic
静态 + LLM 语义分析，平衡速度与深度，适合日常开发验证。
--level deep
LLM 深度分析，全面理解代码意图，精度最高但速度较慢。

LLM 配置（可选）

如需语义验证，设置环境变量：

export SILICONFLOW_API_KEY=your_key_here   # 推荐，支持 DeepSeek-V3
# 或
export DEEPSEEK_API_KEY=your_key_here
# 或
export OPENAI_API_KEY=your_key_here
# 本地 Ollama
export OLLAMA_BASE_URL=http://localhost:11434

输出格式

使用 --format 指定输出格式：

python -m src.cli.main verify -f app.py -r req.txt --format json
python -m src.cli.main verify -f app.py -r req.txt --format markdown

VSCode 插件

安装 VSCode 插件，Ctrl+Shift+V 一键验证当前文件：

cd vscode-extension
code --install-extension intentguard-*.vsix

常见问题

Q: 提示 "Repository not found"

仓库不存在，需要先在 GitHub 上创建仓库后再推送。

Q: LLM 验证返回 401

API Key 无效或额度用完，检查环境变量是否正确设置。

Q: JS/Go 分析结果不准确

JS/Go 使用正则解析，精度不如 Python AST。对于关键验证建议用 Python 重写核心逻辑。
