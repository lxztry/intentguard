CLI 参考

完整命令行接口文档

verify — 验证代码

核心命令，验证代码是否满足需求约束。

verify
python -m src.cli.main verify -f path/to/code.py -r requirements.txt

必选参数

-f, --file待验证的代码文件路径
-r, --requirements需求文件路径（.txt 或 .md）

可选参数

-c, --code直接传入代码字符串（跳过文件读取）
--level验证级别：static | semantic | deep（默认 static）
--format输出格式：text | json | markdown（默认 text）
--output输出到文件路径（默认 stdout）
--lang语言：python | javascript | go（默认 auto 检测）
--llm强制使用指定 LLM：siliconflow | deepseek | openai | ollama
--no-cache禁用 LLM 响应缓存

完整示例

# 基本验证
python -m src.cli.main verify -f app.py -r req.txt

# 指定验证级别 + 输出格式
python -m src.cli.main verify -f app.py -r req.txt --level semantic --format json --output report.json

# 直接传入代码字符串
python -m src.cli.main verify -c "def send_sms(): sms_service.send()" -r "必须发送短信通知"

# 强制使用 DeepSeek LLM
python -m src.cli.main verify -f app.py -r req.txt --llm deepseek --level semantic

# 禁用缓存 + 指定语言
python -m src.cli.main verify -f app.py -r req.txt --no-cache --lang python

parse — 解析需求

仅解析需求文件，查看 IntentGuard 如何理解你的需求（不执行验证）。

parse
python -m src.cli.main parse -r requirements.txt

analyze — 分析代码

仅分析代码结构，不验证需求（查看 AST/函数/调用提取结果）。

analyze
python -m src.cli.main analyze -f path/to/code.py

返回码

0验证通过（所有约束满足）
1验证失败（有约束未满足）
2错误（文件不存在、解析失败等）
